04

Receive retouched assets, review & approve

Crawl
Today — current process

Once Pixelz completes editing, the team downloads retouched assets (three versions per product: natural, white, and transparent backgrounds) and re-uploads them into Frame.io. A Jira review ticket is then raised with the Frame.io link manually pasted in. 36+ Jira automations route the ticket to the correct PD/merchandising reviewer. Reviewers markup in Frame.io. If PD requests changes, a separate Jira ticket is created for internal retouching (human-level edits beyond Pixelz). Once approved, the team comments approval in Jira and moves assets to the server (backup), Elastic (wholesale), and Bynder (DAM).

01Download retouched assets from Pixelz (3 versions each)Manual
02Re-upload to Frame.ioManual
03Raise Jira review ticket + paste Frame.io linkManual
04Jira routes to PD/merch reviewer (36+ automations)Auto
05Reviewer marks up and provides feedback in Frame.io
06If changes: Jira ticket for internal retoucherManual
07Comment approval in Jira, move to server + Elastic + BynderManual
Pain points
×Manual download from Pixelz and re-upload to Frame.io for every batch
×Frame.io link must be manually pasted into every Jira review ticket
×Reviewers switch between Jira (routing) and Frame.io (feedback) constantly
×Frame.io bulk updates don’t stick; support took 5+ weeks to resolve issues
×Post-approval distribution to server, Elastic, and Bynder is manual
On Air — proposed workflow

Air receives retouched assets from Pixelz via API and auto-versions them by unique ID. An Air link is automatically populated in the Jira review ticket. Reviewers markup directly in Air with annotations, version comparisons, and kanban status tracking. If changes are needed, the internal retoucher edits via Air’s desktop sync app with auto-versioning. Approvals auto-update Jira. All 36+ Jira routing automations stay untouched.

01Pixelz API returns edited files to Air, auto-versioned by uidAuto
02Air link auto-populated in Jira review ticketAuto
03Jira routes to PD/merch reviewer (36+ automations unchanged)Auto
04Reviewer marks up directly in Air
05If changes: retoucher edits via Air desktop sync, auto-versioned
06Approval auto-updates Jira + triggers distributionAuto
Wins
✓Pixelz API return eliminates manual download and re-upload
✓Air link auto-synced to Jira — no more manual link pasting
✓Markup, versioning, and kanban all in one place — no tool-switching
✓Desktop sync app lets internal retouchers edit in any tool with auto-versioning
✓Dedicated Slack channel + engineering support — hours, not weeks

DAM consolidation, variants & distribution

Run
Today — current process

Approved assets go to three destinations: the local server (backup), Elastic (wholesale platform), and Bynder (DAM). From Bynder, assets flow via Datago to Akeneo (PIM) to Shopify. Naming conventions (e.g., “_ECOMM”, “_hero”, “_secondary”) trigger automated cropping specs and version generation. Bynder hosts campaign landing pages with hero imagery, secondary imagery, and collection assets. 98% team satisfaction, but versioning bugs and metadata limits persist.
